CC6.1 — Logical Access Controls

Control Objective: The entity implements logical access security measures to protect against unauthorized access to information assets.

Control Description AI DraftEdited

Cloudvault Technologies maintains a centralized identity management system (Okta) that enforces role-based access controls across all production systems. Access provisioning follows a least-privilege model with approval workflows routed through the hiring manager and IT Security team.

Multi-factor authentication (MFA) is required for all employees accessing production environments, including VPN connections and cloud management consoles. SSO is enforced for all SaaS applications in the technology stack.

Test Procedures Edited

1. Obtained and inspected the Okta access policy configuration, confirming MFA enforcement is active for all user groups assigned to production-tier applications.

2. Selected a sample of 25 user access provisioning events from the audit period. Traced each to an approved access request ticket in Jira, confirming manager and IT Security approval prior to access grant.

3. Inspected the quarterly access review report for Q3 2025, verifying that all accounts were reviewed by system owners and inappropriate access was revoked within the 5-day SLA.

Test Results

MFA enforcement verification Pass — No exceptions
Access provisioning sample (25 events) Pass — 25/25 compliant
Quarterly access review (Q3) 1 exception noted

Conclusion AI DraftEdited

Based on the procedures performed, CC6.1 is operating effectively with one exception noted. During the Q3 access review, one service account retained elevated privileges beyond the approved period due to an incomplete deprovisioning ticket. The account was remediated within 8 days of identification. Given the limited scope and prompt remediation, this exception does not affect the overall control effectiveness conclusion.
