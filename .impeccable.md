# Design Context — Axiom

> Generated following impeccable design methodology. This file is the design system source of truth for all UI generation.

## Product

**Axiom** — an AI-native audit engagement platform for mid-market CPA firms (20–60 staff). Replaces spreadsheets, shared drives, and manual workflows with structured engagement management, AI-assisted control testing, and real-time collaboration.

## Users

- **Primary:** Audit professionals — partners (15+ years), managers (8–15 years), staff auditors (1–8 years). Daily users during busy season (Jan–Apr, Jul–Oct). Desktop-primary (1440px+ monitors). Power users who value keyboard shortcuts and information density.
- **Secondary:** Clients — CFOs, controllers, IT managers. Infrequent users who arrive via tokenized email links. Non-technical, task-focused. Need plain language and minimal cognitive load.

## Brand Voice

Three words: **sharp, clinical, confident.**

Physical object: A well-designed compliance report printed on heavy stock — precise typography, clear hierarchy, no decoration. The kind of document that communicates "we know what we're doing" without saying it.

## Aesthetic Direction

Clean, information-dense but not cluttered. Data tables, status indicators, and structured forms are the primary UI patterns. The platform handles sensitive financial and compliance data — the design should convey reliability and professionalism.

**Reference points:** Linear (information density, keyboard-first), Notion (clean surfaces, content focus), Figma (professional tool feel). NOT Salesforce (too heavy), NOT Oracle (too dense), NOT generic SaaS dashboards with hero gradients.

**Anti-references:** Gradient hero sections. Glassmorphism. Cards with heavy drop shadows. Decorative illustrations. Anything that looks like a marketing template.

## Typography

### Font Selection

**Primary (headings + UI):** Plus Jakarta Sans — modern geometric humanist, professional without being generic, excellent weight range (400–800), good tabular number support. Available via Google Fonts.

**Monospace (data, code):** JetBrains Mono — for tabular financial data, code references, and fixed-width content. Available via Google Fonts.

**Why Plus Jakarta Sans:** It's sharp and geometric without being cold. It has the precision of a technical tool font but enough warmth to not feel sterile. It's not on the overused defaults list (Inter, Roboto, Open Sans, DM Sans, Montserrat). Its tabular figures align financial data correctly.

### Type Scale (Major Third — 1.25 ratio)

| Token | Size | Weight | Use |
|-------|------|--------|-----|
| `--text-xs` | 0.64rem (10.2px) | 500 | Badges, micro-labels |
| `--text-sm` | 0.8rem (12.8px) | 400 | Captions, metadata, table secondary |
| `--text-base` | 1rem (16px) | 400 | Body text, table cells, form inputs |
| `--text-lg` | 1.25rem (20px) | 600 | Section headings, card titles |
| `--text-xl` | 1.563rem (25px) | 700 | Page headings |
| `--text-2xl` | 1.953rem (31.3px) | 700 | Hero headings (signup, confirmation) |
| `--text-3xl` | 2.441rem (39px) | 800 | Marketing/landing only |

**Line heights:** Body 1.5, headings 1.2, tight UI elements 1.3.
**Letter spacing:** Headings -0.02em, body 0, small caps 0.05em.
**Measure:** `max-width: 65ch` for prose. No limit for data tables.
**Tabular nums:** Always `font-variant-numeric: tabular-nums` on financial data, status counts, dates.

### Common Components

- **Data tables:** Horizontal borders only (`neutral-200`), header row `neutral-50` bg, `text-sm` for headers (uppercase, `neutral-400`, weight 600, letter-spacing 0.05em), `text-base` for cells
- **Status badges:** Pill-shaped, semantic color bg (light), semantic color text (dark). Padding `space-1` vertical, `space-2` horizontal. `text-xs` weight 600.
- **Cards:** `neutral-0` bg, 1px `neutral-200` border, 8px border-radius, `space-6` padding. No heavy shadows.
- **Buttons — primary:** `primary-500` bg, white text, 6px border-radius, `space-3` vertical / `space-5` horizontal padding, weight 600
- **Buttons — secondary:** transparent bg, `neutral-200` border, `neutral-700` text
- **Form inputs:** 40px height, 1px `neutral-200` border, 6px radius, `space-3` horizontal padding, `neutral-300` placeholder
- **Progress bars:** 8px height, `neutral-200` track, `primary-500` fill, 4px radius
- **AI content marker:** Left border 3px `AI` base color, `AI` light bg, small "AI Generated" badge in `text-xs`

## Quality Gates

Before considering any screen complete:

1. No gradient text fills
2. No glassmorphism
3. No AI-palette monoculture (every screen same blue gradient)
4. No pure gray — all neutrals are tinted
5. No pure black (#000)
6. Tabular numbers on all financial data
7. Status badges use semantic colors, not arbitrary ones
8. AI-generated content always has a visual marker
9. Data tables have 5–10 realistic rows
10. WCAG AA contrast on all text (4.5:1 body, 3:1 large)
